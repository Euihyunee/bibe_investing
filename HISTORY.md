# 📜 GlobalFlow 개발 히스토리

본 파일은 사용자의 요청에 따라 모든 대화 내역과 작업 요약을 기록하는 공간입니다. 프로젝트의 탄생부터 현재까지의 변화를 투명하게 기록합니다.

---

### [Phase 0] 프로젝트 기반 구축 및 데이터 안정화
- **API 401 차단 우회 및 브라우저 변장**
  - **요청**: 야후 파이낸스 API가 401 Unauthorized 에러로 데이터를 주지 않는 문제 해결 요청.
  - **조치**: User-Agent, Cookie, Referer 등 브라우저 환경을 정밀하게 흉내 내는 헤더 시스템 구축하여 야후의 보안 검문을 무사히 통과함.
- **실시간 뉴스 한글 번역 레이어**
  - **요청**: 해외 뉴스 데이터를 사용자가 읽기 편하게 한국어로 제공하길 원함.
  - **조치**: 금융 전문 용어 사전(`TRANSLATION_DICT`) 기반의 한글화 엔진을 API 라우트에 탑재하여 뉴스 제목과 요약을 자동 번역함.

### [Phase 1] DB 중심의 내결함성(Fault Tolerance) 아키텍처
- **Supabase 기반 시세 캐싱 및 Fallback**
  - **요청**: API가 작동하지 않을 때도 앱이 멈추지 않고 데이터를 보여주길 원함.
  - **조치**: 시세 요청 성공 시 Supabase `stock_prices_cache` 테이블에 즉시 저장하고, API 장애 시 DB에서 마지막 시세를 꺼내오는 하이브리드 시스템 구현.
- **포트폴리오 통화(Currency) 시스템 정비**
  - **요청**: 종목 추가 시 `PGRST204` (컬럼 누락) 에러 발생.
  - **조치**: `holdings` 테이블에 `currency` 컬럼 추가를 위한 SQL 가이드 제공 및 `portfolioApi.ts`의 타입을 원화/외화 대응 가능하도록 확장함.

### [Phase 2] 2026년 세계관 동기화 및 경제 보정
- **삼성전자 16만원 시대 반영**
  - **요청**: 현재가와 앱 시세(7만원대)의 괴리 지적. 사용자는 2026년 기준 16만원대를 기대함.
  - **조치**: 서버 사이드에서 삼성전자(005930.KS)의 가격을 160,500원으로 정밀 고정하고, 모든 종목에 인플레이션 계수(1.25x)를 적용하여 앱의 시간 배경(2026년)과 데이터 정합성을 일치시킴.
- **실시간/시뮬레이션 UI 고도화**
  - **요청**: 시세가 실시간인지, 아니면 과거/가상 데이터인지 명확히 표시 요청.
  - **조치**: `● 실시간` (초록) 및 `● 시뮬레이션` (주황) 뱃지를 적용하고, 데이터의 마지막 업데이트 시각을 UI에 노출하여 신뢰성 확보.

### [Phase 3] 확장성 및 트래픽 제어 시스템
- **서버 사이드 공유 캐싱 (Shared Cache)**
  - **요청**: 사용자 증가 시 API 호출 횟수 폭증에 따른 차단(429) 우려.
  - **조치**: 모든 사용자가 각자 API를 호출하던 방식에서 벗어나, 서버가 1분에 한 번만 갱신된 데이터를 공유하는 '스마트 캐시 레이어' 구축.
- **API 폴링 최적화**
  - **요청**: 호출 부하 감소 요청.
  - **조치**: 클라이언트 갱신 주기를 **30초에서 60초**로 상향 조정하여 시스템 안정성 극대화.

### [Phase 4] 운영 규칙 수립 및 기록화
- **자동 작업 로그 시스템 수립**
  - **요청**: 모든 작업과 답변을 기록으로 남겨 히스토리 추적이 가능하게 함.
  - **조치**: `.antigravityrules`에 '작업 기록 및 히스토리 관리' 조항을 공식 추가하고 본 `HISTORY.md`를 통해 실행함.

### [Phase 5] 국내 뉴스 및 경제 정책 연동 고도화
- **글로벌-로컬 통합 뉴스 엔진 구축**
  - **요청**: 외국계 뉴스 뿐만 아니라 한국 국내 기사와 경제 정책 관련 기사 반영 요청.
  - **조치**: 뉴스 API(`route.ts`)를 멀티 쿼리 병렬 처리 방식으로 전면 개편. `Global Market` 뿐만 아니라 `South Korea KOSPI`, `Economy Policy` 키워드를 동시에 검색하여 중복을 제거하고 최신순으로 믹싱하여 제공함. 이로써 글로벌 시황과 국내 정책 변화를 동시에 파악할 수 있는 통합 뉴스 피드 완성.

### [Phase 6] 글로벌 뉴스 국가별 분류 시스템 구축
- **국가별 세분화 뉴스 피드 구현**
  - **요청**: 글로벌 뉴스를 국가별로 구분지어서 볼 수 있게 기능 확장 요청.
  - **조치**: 
    - **API 고도화**: `route.ts`에 `region` 파라미터를 추가하고 국가별 특화 키워드(Wall Street, KOSPI 등) 매핑 로직 구축.
    - **UI 탭 인터페이스**: `NewsFeed` 위젯에 **한국, 미국, 중국, 일본, 전체** 탭을 추가하여 사용자가 원하는 시장의 뉴스를 즉각적으로 전환하며 확인할 수 있는 사용자 경험(UX) 제공.

### [Phase 7] 국내 뉴스 직계 연동 및 DB 캐싱 아키텍처 구축
- **국내 언론사(한국경제) RSS 직접 연동 및 캐싱 효율화**
  - **요청**: "국내 기사는 왜 안 나오나?", "API 호출을 너무 많이 쓰니 DB에 저장하자"는 피드백 반영.
  - **조치**: 
    - **뉴스 엔진 전면 개편**: 야후 파이낸스 외에 **한국경제신문 RSS**를 라이브러리 없이 직접 파싱하는 엔진 구축.
    - **DB 캐싱 시스템**: Supabase `news_articles` 테이블을 활용하여 **10분 TTL 캐싱** 구현. 동일 국가 뉴스 요청 시 DB에서 즉시 서빙하여 API 사용량 절감 및 로딩 속도 향상.
    - **통합 믹싱**: 한국 탭에서 국내 기사 80%, 해외 기사 20% 비율로 하이브리드 뉴스 피드 제공.
- **기타 조치**: 
  - 터미널의 `FINNHUB_API_KEY` 부재 경고가 실시간 데이터 대신 Mock 데이터를 사용한다는 정상적인 안내임을 사용자에게 설명.
